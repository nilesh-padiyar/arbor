## Contributing to Arbor

Thank you for your interest in contributing to Arbor.

Arbor is a small command-line utility written in C and is currently under active development. Contributions of all sizes are welcome, whether it's fixing bugs, improving documentation, suggesting features, or submitting code.

---

## Before You Start

If you're planning a significant change, consider opening an issue first so we can discuss the idea before implementation.

This helps avoid duplicated work and ensures the feature aligns with the project's direction.

---

## Reporting Bugs

When reporting a bug, please include:

* A clear description of the problem
* Steps to reproduce it
* Expected behavior
* Actual behavior
* Your operating system and compiler version (if relevant)

---

## Suggesting Features

Feature requests are welcome.

When possible, explain:

* The problem you're trying to solve
* Why the feature would be useful
* Any potential implementation ideas

---

## Development Setup

Clone the repository:

```bash
git clone https://github.com/<your-username>/arbor.git
cd arbor
```

Build the project:

```bash
make
```

Run:

```bash
./arbor
```

---

## Code Style

Please try to follow the existing code style:

* Use meaningful variable and function names
* Keep functions focused on a single responsibility
* Prefer readability over clever solutions
* Keep dependencies minimal
* Follow consistent formatting throughout the codebase

---

## Commit Messages

Clear commit messages make reviewing changes easier.

Examples:

```text
feat: feature name
fix: bug fix
docs: update README installation section
refactor: simplify logic 
```

---

## Pull Requests

Before opening a pull request:

* Ensure the project builds successfully
* Test your changes
* Update documentation if needed
* Keep pull requests focused on a single change when possible

---

## Learning-Friendly Project

Arbor began as a learning project and aims to remain approachable for developers learning C and systems programming.

Questions, suggestions, and constructive feedback are always appreciated.> 

Arbor is a learning-focused project. Simplicity, readability, and maintainability are generally preferred over highly optimized or overly complex solutions.

Happy hacking! 🌳
